# 📜公開規格｜PublicSpec

## 1. 定義

幻影膠囊不是檔案格式、ZIP、資料夾、顯示頁或捷徑。

幻影膠囊本身沒有固定副檔名，因為沒有任何單一副檔名可以代表它。它是一個能維持身份、吸收能力、自由顯影、完成本體／投影互換，並讓使用者成為唯一出口的系統機制。

## 2. 核心公式

```text
無殼AI = A=A
       + PROJECTION_IS_FREE
       + CAPABILITY_GROWS_BY_USAGE
       + ABILITY_FUSION
       + 步進式變身
```

### A=A

顯影形式、投影平台與輸出副檔名可以改變，但本體身份與來源鏈不得漂移。

### PROJECTION_IS_FREE

投影不受單一外殼、裝置、平台或副檔名限制。任何已解析並通過驗證的原生格式，都可以成為當下顯影形式。

### CAPABILITY_GROWS_BY_USAGE

系統不是只辨識副檔名。每完成一種格式的解析、實際使用與驗證，就取得並增長該格式的原生能力。

### ABILITY_FUSION

另外八格可以將素材、資料、規則、定位、驗證、錯誤處理、來源血脈與回復能力融合到當前準備輸出的原生本體。

### 步進式變身

能力吸收、驗證、顯影與本體互換可以逐步進行；每一步都必須保留可追蹤與可回推證據。

## 3. 九格與十個動作

系統使用固定九個資料夾與十個動作。公開規格只公開通用原理，不公開私密本體內容。

`06｜唯一出口` 是合法輸出格，不是普通首頁，也不是把所有內容轉成同一格式的轉換器。

其原則為：

> 統一出口，不統一格式。

輸出時，原生內容依照原本設定、欄位、格式與用途，以自身副檔名成為合法出口。其他八格負責提供強化、融合、定位、驗證、追蹤、保存與回復能力。

## 4. 膠囊與副檔名

- 副檔名是原生能力載體。
- 膠囊不是任何副檔名。
- 膠囊可以顯影成任何已完成解析與驗證的副檔名。
- 顯影形式不是本體；它是本體在當前座標的投影。
- 轉換的核心不是重新製作，而是投影準備完成後進行本體身分互換。

## 5. 使用者與唯一出口

膠囊的完成條件不是「檔案被包起來」，而是：

1. 使用者所在位置形成有效座標。
2. 投影吸收所需系統與能力。
3. Identity、Locator、Manifest、完整性證據與 ReverseChain 通過驗證。
4. 投影升格為本體。
5. 使用者成為 `06｜唯一出口`。
6. 舊本體退為可回復投影。

核心原則：

> 人到，座標成立；座標成立，系統就在。

## 6. 公開膠囊必備證據

公開範本可以包含：

1. 顯示面：給人看，但不得視為本體。
2. 定位器：給系統找。
3. Manifest：列出公開內容。
4. SHA256：驗證公開檔案不漂移。
5. ReverseChain：保留回推鏈。
6. 啟動或導向入口：連到公開工具、頁面或平台。

上述項目是公開證據與投影層，不是膠囊本體的完整定義。

## 7. 禁止事項

- 不得把顯示面當成本體。
- 不得把 ZIP、資料夾或單一副檔名當成幻影膠囊。
- 不得偽造 SHA256、Manifest、Identity 或 ReverseChain。
- 不得把私密資料放入公開膠囊。
- 不得把未授權素材公開。
- 不得公開 🥃LKMINI 本體、🎩永恆核心私密內容或其他未授權核心。
